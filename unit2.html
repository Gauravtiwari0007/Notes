Features of C#

Definition

C# is a modern, object-oriented, type-safe programming language developed by Microsoft for the .NET platform.

Explanation

Some features include:

  • Simple and easy to learn
  • Object-Oriented
  • Type-Safe (prevents type errors)
  • Rich Standard Library
  • Automatic Memory Management (Garbage Collection)
  • Cross-platform support

C# Compilation and Execution

Definition

Compilation is the process of converting source code into Intermediate Language (IL), which is executed by the CLR (Common Language Runtime).

Explanation

Steps:

  1. C# Compiler (csc.exe) converts code into IL.
  2. IL is stored in an assembly (.exe or .dll).
  3. CLR‚Äôs Just-In-Time (JIT) compiler converts IL to machine code.
  4. OS executes the program.
Program.cs ‚Üí csc Program.cs ‚Üí Program.exe ‚Üí CLR ‚Üí Output
Analogy: Think of it like translating a book from English ‚Üí an intermediate ‚Äúsymbol language‚Äù ‚Üí then into Hindi when you read it.
// Save this as Program.cs
using System;
class Demo {
    static void Main() {
        Console.WriteLine("Compilation Demo");
    }
}
// Compile: csc Program.cs
// Run: Program.exe
Exam Notes: Compilation in C# involves 2 steps ‚Äì compiling to IL and JIT compiling to native code at runtime.

Creating and Using a DLL

Definition

A DLL (Dynamic Link Library) is a reusable library file that contains compiled code, which can be used by multiple applications.

Explanation

  1. Write a class library in C#.
  2. Compile using csc /target:library File.cs ‚Üí generates .dll.
  3. Reference DLL in other projects using using statement.
Analogy: A DLL is like a ‚Äúshared kitchen‚Äù in a hostel. Everyone can use it instead of each room having its own kitchen.

    public int Add(int a, int b) {

    }
}
// Compile: csc /target:library MyLib.cs
// Generates MyLib.dll

Value Types and Reference Types

Definition: Value types store data directly, while reference types store a reference (address) to the actual data.

string s = "Hi";  // Reference type (heap)
Analogy: Value type = writing a number directly on paper. Reference type = writing the page number where the number is written.
Exam Notes: Value = stack, Reference = heap. Structures are value types; classes are reference types.

Boxing and UnBoxing

Definition: Boxing converts a value type into an object (reference type). Unboxing extracts the value type from the object.

int num = 123;       // Value
object obj = num;    // Boxing
int x = (int)obj;    // Unboxing
Analogy: Boxing = putting a gift (value) into a box (object). Unboxing = opening the box to take it out.
Exam Notes: Boxing = implicit, Unboxing = explicit.

The ‚Äòobject‚Äô base class in .NET

Definition/Concept

In .NET, System.Object (or simply object) is the root base class for all types. Every class or struct in C# inherits from it automatically.

Detailed Explanation

  • Inherited Methods: ToString(), Equals(), GetHashCode(), GetType().
  • Implicit Inheritance: class MyClass { } is same as class MyClass : object { }.
  • Overriding: You can override ToString(), Equals(), etc., in your own class.
  • Boxing: Value types stored in object are boxed.
Analogy: Object is like the ‚Äúcommon ancestor‚Äù in a family tree ‚Äî all types in .NET come from it.

Diagram

           Object
           /   \
        Int32  String
           \
          Double

Code Example

object o1 = 123;            
object o2 = "Hello";        
Console.WriteLine(o1.ToString());  // "123"
Console.WriteLine(o2.ToString());  // "Hello"

class Person {
    public string Name;
    public override string ToString() => $"Person: {Name}";
}
Person p = new Person { Name = "Alice" };
Console.WriteLine(p.ToString());  // "Person: Alice"
Exam Notes:
- Root of .NET type system.
- Provides methods like ToString(), Equals(), GetHashCode().
- Can store any type using object.
- Custom classes often override ToString() and Equals().

Equals() vs ==

Definition/Concept

== is an operator that can be overloaded. .Equals() is a virtual method inherited from Object. By default, both check reference equality, but types like string override them to compare values.

Detailed Explanation

  • Value types: == and Equals() both compare values.
  • Strings: both compare contents (case-sensitive).
  • Reference types (default): == and Equals() compare references unless overridden.
Analogy: == is like asking ‚ÄúDo we point to the same toolbox?‚Äù while Equals() (when overridden) asks ‚ÄúDo both toolboxes contain the same tools?‚Äù

Code Example

int i = 5, j = 5;
Console.WriteLine(i == j);       // True
Console.WriteLine(i.Equals(j));  // True

string s1 = "test";
string s2 = String.Copy(s1);
Console.WriteLine(s1 == s2);       // True
Console.WriteLine(s1.Equals(s2));  // True

class Person { public string Name; }
Person p1 = new Person { Name="Bob" };
Person p2 = new Person { Name="Bob" };
Console.WriteLine(p1 == p2);       // False
Console.WriteLine(p1.Equals(p2));  // False
Exam Notes:
- ==: Operator, can be overloaded.
- Equals(): Virtual method, can be overridden.
- Strings override both for content comparison.
- Custom classes should override both consistently.

String vs StringBuilder

Definition/Concept

string: immutable text type.
StringBuilder: mutable sequence of characters for efficient modifications.

Detailed Explanation

String creates new objects on each modification. StringBuilder modifies content in-place, which is faster in loops or repeated changes.

Analogy: string is like writing on stone ‚Äî you must make a new stone for each change. StringBuilder is like a whiteboard ‚Äî you can erase and rewrite.

Code Example

// String (immutable)
string s = "Hello";
s += " World";          // New string created
s = s.Replace('o','a'); // "Hella Warld"

// StringBuilder (mutable)
var sb = new StringBuilder("Hello");
sb.Append(" World");
sb.Replace('o','a');
Console.WriteLine(sb.ToString());
Exam Notes:
- string: immutable, new object each change.
- StringBuilder: mutable, efficient in loops.
- Use StringBuilder for performance in repeated concatenations.

Default & Named Parameters

Definition/Concept

Default parameters: allow omitting arguments with predefined values.
Named parameters: allow specifying arguments by name in any order.

Code Example

void SendEmail(string to, string subject="No subject", bool isHtml=false)
{
    Console.WriteLine($"To: {to}, Subject: {subject}, HTML: {isHtml}");
}

SendEmail("user@test.com");                       
SendEmail("user@test.com", subject: "Hello!");
SendEmail("user@test.com", isHtml:true, subject:"Hi");
Analogy: Like ordering food with default sides. You can just say ‚ÄúSteak‚Äù (defaults applied) or specify ‚ÄúSteak, with Salad=false, Potatoes=true‚Äù by naming arguments.
Exam Notes:
- Default parameters reduce overloads.
- Named parameters improve readability.
- Named args can be reordered.
- Rule: defaults must come last in signature.

Parse, TryParse, and Convert

  • Parse() ‚Üí Converts string to value type (throws exception if invalid).
  • TryParse() ‚Üí Safe conversion, returns bool.
  • Convert ‚Üí Converts between types, handles null.
int a = int.Parse("123");              // Parse
bool ok = int.TryParse("abc", out int b); // TryParse
double d = Convert.ToDouble("45.6");   // Convert
Analogy: Parse = strict teacher (throws error). TryParse = friendly teacher (gives result + success/fail). Convert = multipurpose teacher (handles nulls too).
Exam Notes: TryParse ‚Üí best for safe conversions without exceptions.
